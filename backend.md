1. Два ORM в одном проекте — это компромисс. Фронт использует Drizzle для калькулятора, бекенд — Prisma. Не критично (они не пересекаются по
  таблицам), но при расширении стоит мигрировать всё на один ORM.

  2. Деплой — NestJS как long-running сервер не подходит для Vercel (serverless). Для прода нужен отдельный хостинг (Railway, Fly.io, VPS). Но
  для текущего этапа разработки это нормально.

  3. Нет кеширования — сейчас revalidate: 60 на фронте, но на бекенде нет Redis/in-memory cache. Для 3 постов пока неважно, но при
  масштабировании стоит добавить @nestjs/cache-manager.
